# 红圈验证 · 关卡投稿指南

想把你的整活关卡塞进 [Circle.html](./Circle.html) 的轮播池？欢迎。  
**但请先看完这一页。**

---

## 为什么有门槛

这个池子是随机抽 5 关播给所有访客的。  
一张没人 get 的梗图、一张盗版二次元截图、一张 NSFW —— 都会让别人尴尬。  
所以下面的清单里**任何一项不达标，PR 直接 close，不解释。**

---

## 五分钟流程

> 前提：你会用 git，会发 PR。如果不会，[先去学一下](https://docs.github.com/zh/get-started)再回来。  
> 拒绝在 issue 里贴图求"哪位大佬帮我合一下"。

1. **Fork** [`CS-LX/Inhuman`](https://github.com/CS-LX/Inhuman) → clone 到本地。
2. **打开 [`Editor.html`](./Editor.html)**（双击或本地服务器），左上角填：
   - 作者名 = **你的 GitHub 用户名**（不是昵称、不是花名）
   - 作者链接 = `https://github.com/<你的用户名>`（可选）
   - Pack 名 = 一个**只含小写字母/数字/连字符**的英文短名，例如 `my-cats`
3. **左侧路径前缀改成 `imgs/<你的用户名>/`**（比如 `imgs/octocat/`）。
4. 上传图片对、拖红圈、滚轮调半径、加入列表。**至少 3 组，至多 20 组。**
5. **导出 pack JSON**（按钮就在左下角，文件名会自动 = `<pack>.json`）。
6. 在仓库里：
   ```
   Circle/imgs/<你的用户名>/    ← 把所有 PNG 放进来
   Circle/configs/<pack>.json   ← 把刚才导出的 JSON 放进来
   Circle/configs/manifest.json ← 在 "packs" 数组末尾加一行 "<pack>.json"
   ```
7. `git commit -m "feat(Circle): add pack <pack> by <你的用户名>"` → push → 在 GitHub 上发 PR。

## PR 提交清单（直接复制进 PR 描述）

```markdown
### Pack 信息
- Pack 名: 
- 作者(GitHub): @
- 关卡数: 

### 整活思路（≥ 30 字，认真写，让审核能笑出来）


### 我已确认
- [ ] 至少 3 组关卡，每组都用 Editor 拖过红圈、不是手填坐标
- [ ] 所有图片都在 `Circle/imgs/<我的用户名>/` 下，没塞到别人的目录
- [ ] 每张 PNG ≤ 2MB，最短边 ≥ 640px
- [ ] 已在 `configs/manifest.json` 的 packs 数组里注册了我的 pack
- [ ] 没有用任何明星 / 知名 IP / 公司 logo / 他人原创未授权素材
- [ ] 没有 NSFW、政治敏感、人身攻击内容
- [ ] 我清楚我的 GitHub 用户名会作为作者署名公开显示
```

**清单里任何一项你没勾、或者勾了但实际不符 —— close。**

## 会被秒拒的 PR

- 图片放在 `imgs/` 根目录或 `imgs/default/` 下（**不许污染别人的命名空间**）
- 只交了 1–2 组关卡
- JSON 是手写的，target 坐标看着像随便填的
- 整张图就是一个表情包 / 一句"梗"，没有"找不同/找位置"的可玩性
- PR 描述只有标题没正文 / 整活说明少于 30 字 / 写"6"、"哈哈哈"、"看就完事了"
- 用了别人原创的图（包括但不限于：游戏截图、动漫角色、明星照片、网图）
- 起了一个像 `test` / `123` / `mypack` 这种敷衍的 pack 名

## 关于"创意"的一点话

最好的红圈关是这样的：
- 一眼看上去图里东西很多，但你**真的能找到**那个目标
- 找到的瞬间会让人"哦——卧槽——"

最差的红圈关是这样的：
- 图全是马赛克 / 纯色 / 抽象到没人能找
- "找不到才是整活"——**不是**，那是你不会做关卡

如果你只是想一时爽，发条动态比发 PR 快多了。这里只收能让陌生人也乐出来的东西。
